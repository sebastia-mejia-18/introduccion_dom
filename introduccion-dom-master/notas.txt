Métodos de los arreglos:
    map() -> Iteración que manipula todo el arreglo el cual retornará un nuevo arreglo
    filter() -> Iteración que filtra el contenido de un arreglo bajo una condición retornando un nuevo arreglo
    find() -> Iteración que busca un único elemento dentro de un arreglo y retorna ese elemento
    some() -> Iteración que busca un único elemento dentro de un arreglo y retorna true o false
    push() -> Método que agrega un elemento a la ultima posición de un arreglo
    pop() -> Método que elimina la ultima posición de un arreglo
    shift() -> Método que elimina la primera posición de un arreglo
    unshift() -> Método que agrega un elemento a la primera posición de un arreglo
    splice() -> Método que elimina un elemento de una posición n del arreglo (n puede ser cualquier valor)
    forEach() -> Lo mismo que el map pero sin retorno de arreglo

    Métodos Mutables -> Modifica el arreglo original

    Métodos no Mutables -> No modifica el arreglo original
        map()

Para los input la propiedad que permite ver y manipular el contenido es la value
Una estructura de datos que puede almacenar cualquier otro tipo de dato: Son una agrupación de un par de datos: (llave y valor)

El DOM es manipulado por medio de Eventos
Eventos de usuario -> Toda acción que realiza un usuario dentro de la apliación: click, doble click, scroll, presionar una tecla, etc.
Eventos de documento -> Toda acción que se realiza por debajo y no es visible para el usuario

Los eventos son funciones o acciones que se pueden definir desde el html o desde el JavaScript, y siempre serán ejecutados por medio de funciones

Funciones -> Bloque de código encapsulado que se puede llamar donde quiera y ejecutará una acción definida
Toda función trabaja en dos tiempos: Su definición y su llamado o invocación

    Funciones Regulares
        Declarativas -> Se almacenan en el alcance global de la aplicación (hoisting)
        De expresión
    Funciones Flecha
        De expresión
    Funciones anónimas -> Es una función que no tiene nombre y se ejecuta solamente donde se definió

    No retorno -> Ejecuta y muestra en la misma función (void)
    Retorno -> Una ejecución que hace una función y que se podrá utilizar después


Eventos predeterminados
    Mouse -> click, doble click, ingreso del mouse, salida del mouse, scroll
        onclick, ondblclick, onmouseenter, onmouseout
    Teclado -> onChange
